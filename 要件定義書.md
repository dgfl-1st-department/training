# アプリケーション要件定義書

## 1. 目的・背景

### 1.1 背景・課題

- 従業員の業務満足度や健康状態を定期的に把握する仕組みがなく、組織の状態を早期に把握しづらい
- 人事・管理部門が部署別・時系列の傾向を把握する手段が限られている

### 1.2 目的

- 従業員の業務満足度および健康状態を定期的に測定し、組織の状態を可視化する
- 管理者が部署別・期間別の集計データを参照し、組織改善の意思決定を支援する

### 1.3 期待される効果

- 組織の状態変化の早期検知
- データに基づく人事・組織施策の検討支援

---

## 2. スコープ

### 2.1 対象（ロール）

| ロール | 説明 |
| --- | --- |
| 従業員 | 回答および自身の回答履歴参照のみ可能 |
| 管理者 | 配下従業員の回答参照、集計ダッシュボードの参照が可能 |
| システム管理者 | アカウント管理、質問管理による設定変更が可能 |

### 2.2 対象外

- 医療的診断や個別面談の管理
- 外部システムとの連携（Google Workspace認証以外のAPI連携・SSO）
- モバイルアプリ（Webアプリケーションのみを対象とする）

---

## 3. 用語定義

| 用語 | 定義 |
| --- | --- |
| 従業員 | 回答を登録・編集する一般ユーザー |
| 管理者 | 質問の管理と集計閲覧を行うユーザー |
| システム管理者 | ユーザー管理、部門管理、質問管理を行うユーザー |
| カテゴリ | 測定項目のグループ |
| 質問 | 測定対象となる設問 |
| 回答 | 従業員が質問に対して入力した5段階評価および自由記述 |
| レーティング | 1 ～ 5 の5段階評価の回答項目 |
| 自由記述 | 自由に記載可能な文章による回答項目 |

---

## 4. 機能要件

### 4.1 回答登録

- 従業員は自分の回答の登録・編集が可能
- **入力形式**: 5段階評価および自由記述に対応
- 回答履歴の閲覧が可能（過去の回答の編集は許可しない）
- 回答日は利用者が選択できず、システムが登録日を付与する
- 回答は同一月内で1回のみ可能とし、当月既に回答済みの場合は回答履歴画面を案内する

### 4.2 回答履歴

- 従業員は自身の回答履歴のみ閲覧できる
- 管理者は、権限に基づき管理対象従業員の回答履歴を閲覧できる
  - 管理対象部署に所属する従業員について参照可能
- 回答履歴に含める内容
  - 回答日、質問、レーティング（5段階）、自由記述
  - 表示順・フィルタ条件は画面設計に従う

### 4.3 質問管理

- システム管理者は質問の登録・編集・削除が可能
- **質問項目**
  - 質問カテゴリ
    - 仕事
    - 人間関係
    - 健康
  - 質問本文
    - 質問内容を記載（文字数上限は画面・バリデーションで定める）
  - 質問の追加・編集・削除
    - レーティング（5段階評価）
    - 自由記述（500文字以内）
    - 上記のどちらかを選択する
  - 質問の公開/非公開の切り替え
  - 質問の並び順の変更
- 管理者は質問管理の閲覧のみ可能

### 4.4 集計

- **閲覧権限**: 管理者およびシステム管理者のみ（従業員向け集計画面は提供しない）
- **表示形式**: 1画面内でグラフと一覧を組み合わせたダッシュボードとする

#### 4.4.1 集計の構成要素

- **推移グラフ（レーティング）**
  - 対象期間内のレーティング平均（または代表値）の推移を、質問ごとに折れ線で表示する
  - 既定の表示単位は月とし、横軸は年月、縦軸はレーティング（1～5）
  - 直近1年間を初期表示とする
- **管理対象一覧**
  - 管理権限の範囲内の従業員をテーブル表示する
  - 表示項目
    - 氏名、拠点、部署、当月の回答内容（レーティング優先、自由記述を末尾に表示）
  - グラフ側でフォーカスした月に合わせ、一覧の表示内容を該当月に連動させる
  - 部署・拠点などで一覧を絞り込める
- **期間の操作**
  - **年度**: 事業年度単位で表示切り替えができること

#### 補足（実装・設計時の整理ポイント）

- グラフの「月次推移」とフィルタの「日／週／月」は、画面上でどちらが主軸か（例: 既定は月次、ドリルダウンで週・日）を画面設計で明示する
- 一覧に出す「回答結果」は、数値（平均）に加え、未回答の有無が分かる表示とすると運用しやすい

#### 検討候補（スコープ外でもよい拡張アイデア）

- 評価の**分布**（1～5の件数ヒストグラム）や、部署間の**比較**ビュー
- **自由記述**の件数・抜粋一覧（個人情報・要約方針は別途定義）
- 集計結果の **CSV エクスポート**（権限・マスキング方針付き）
- **未回答者**のリスト表示とリマインド連携の可否

### 4.5 システム設定

- システム管理者は以下を実施可能
  - アカウント管理
  - 部署管理

#### 4.5.1 アカウント管理

- アカウント情報の閲覧
- アカウントのロール設定
- アカウントの部署設定
- アカウントの拠点設定
- 管理・参照権限に関する前提: 従業員の回答・集計の参照は、**当該従業員の所属部署に対して部署管理（4.5.2）で登録された管理者**に限定される（部署あたり複数の管理者を設定可能）
- アカウントの停止設定（停止アカウントはログイン不可）
- 複数アカウントを選択して一括で設定できる

#### 4.5.2 部署管理

- 拠点情報の閲覧
- 拠点情報の作成・編集・削除
- 部署情報の閲覧
- 部署情報の作成・編集・削除
- 部署の管理者ユーザの設定（複数可）

### 4.6 認証・権限

- 全画面でログイン必須
- **認証方式**: 社内Google Workspaceアカウント認証（OAuth 2.0）
- **セッション**: 有効期限8時間、手動ログアウトを提供

### 4.7 初回アクセス処理

- 初めて本システムにアクセスしたアカウントには初回アクセス画面を表示する
- 初回アクセス画面で以下を入力・選択できるようにする
  - 部署名の選択
  - 拠点の選択

### 4.8 画面一覧

| 画面 | 対象 | 概要 |
| --- | --- | --- |
| ログイン画面 | 全ユーザー | Google Workspace アカウント認証 |
| 初回アクセス画面 | 全ユーザー（初回のみ） | 部署・拠点など初期属性の登録 |
| 回答画面 | 全ユーザー | 5段階評価・自由記述の入力 |
| 回答履歴画面 | 従業員、管理者（閲覧権限内） | 過去の回答一覧（対象範囲はロールに依存） |
| 集計画面 | 管理者・システム管理者 | 部署別・期間別の集計表示 |
| 質問管理画面 | 管理者（閲覧のみ）、システム管理者（編集） | 質問の追加・編集・公開/非公開 |
| システム設定画面 | システム管理者 | アカウント管理・部署管理への導線 |
| アカウント管理画面 | システム管理者 | アカウントの登録・更新・権限設定 |
| 部署管理画面 | システム管理者 | 拠点・部署の登録・更新 |

### 4.9 データ管理

- プライマリキーは一意性が保証される識別子とする。格納形式・採番方式は基本設計書および実装で定める
  - 競合・重複時は登録エラーとして扱い、再試行を促す
- 質問を削除しても、当該質問に紐づく回答データは破棄しない
  - 質問の削除は論理削除とする（履歴・監査の観点で参照可能な状態を維持する）

## 5. 非機能要件

### 5.1 セキュリティ

- 回答データは暗号化して保存（保存時暗号化）
- **監査ログ**: 以下の操作を記録
  - ログイン（成功・失敗）
  - 質問の追加・編集・削除・公開状態変更
  - 集計の閲覧
- 監査ログの改ざん防止措置を講じる

### 5.2 性能

- **レスポンス**: 通常操作で2秒以内に応答すること
- 集計ダッシュボードは、対象データ量に応じて5秒以内を目安とする

### 5.3 可用性

- 社内業務時間帯（想定: 平日 8:00〜20:00）の稼働を想定
- 計画メンテナンスは事前通知の上で実施

### 5.4 運用・保守

- ログの保存期間は別途運用方針で定義
- バックアップは定期取得し、障害時の復旧手順を整備する

---

## 6. 制約・前提

### 6.1 技術・環境

- 社内ネットワークからの利用を想定
- **対応ブラウザ**: 最新2バージョンの Chrome、Edge、Firefox、Safari
- レスポンシブ対応は必須としない（デスクトップ利用を主とする）

### 6.2 業務・組織

- 従業員・管理者は社内Google Workspaceアカウントを保有していることを前提とする
- 部署マスタは他システムまたは手動で管理されていることを前提とする（本システムでの管理は対象外とする場合）

### 6.3 その他

- 本要件定義書に記載のない機能は、原則としてスコープ外とする

# 環境設計書

## 1. 目的
本ドキュメントは、「要件定義書」および「基本設計書」に基づき、アプリケーション開発におけるローカル開発環境および実行環境のベースとなる構成を定義する。

## 2. システム要件と技術スタック
基本設計書に基づき、以下の技術スタックを採用して環境を構築する。

### 2.1 バックエンド
- **言語**: Python 3.12
- **フレームワーク**: FastAPI 0.135+
- **その他主要ライブラリ**: Uvicorn (0.41+), PyMySQL (1.1+), cryptography (46.0+)
- **パッケージ管理**: uv

### 2.2 フロントエンド
- **言語**: TypeScript 5.9.x / React 19.2+
- **ビルドツール**: Vite 8.0+
- **パッケージマネージャ**: npm

### 2.3 データベース
- **RDBMS**: MySQL 8.0+
- **認証プラグイン**: `mysql_native_password` (MySQL 8.0 初期設定用)
- **インフラ**: Docker コンテナとして構築し、データを永続化（ボリュームマウント）

### 2.4 インフラ・コンテナ
- **統合管理**: Docker Compose を用いて、フロントエンド・バックエンド・データベースを同一ネットワーク内で起動
- **ホットリロード**: フロントエンドとバックエンドのコード変更が即座に反映されるよう、ローカルディレクトリをコンテナへマウント
- **留意事項**: 開発は完全にコンテナ内で完結し、ホストOSに Node.js や Python がなくても開発可能

## 4. ネットワークと通信設計

| サービス | ローカルアクセスURL | コンテナ間アクセス（内部通信） |
|----------|---------------------|--------------------------------|
| **Frontend (React)** | `http://localhost:5173` | `frontend:5173` |
| **Backend (FastAPI)**| `http://localhost:8000` | `backend:8000` |
| **Database (MySQL)** | `localhost:3306`        | `db:3306` |

- **CORS設定** (今後実装): バックエンド(FastAPI)側で `http://localhost:5173` からのアクセスを許可する設定を追加する必要がある。

## 5. ローカル開発環境の構築手順

### 5.1 コンテナの立ち上げ
リポジトリ直下で以下のコマンドを実行し、DB、バックエンド、フロントエンドの全コンテナをビルド・起動する。
```bash
docker-compose up -d --build
```
> **確認方法**
> - コンテナのステータス: `docker ps`
> - フロントエンド画面 (React): `http://localhost:5173/`
> - バックエンドAPI (FastAPI): `http://localhost:8000/`
> - APIドキュメント (Swagger UI): `http://localhost:8000/docs`

### 5.2 データベースの永続化・クリーンアップ
DBのボリューム (`db_data`) はホスト側に保持されます。DBを完全にリセットしたい場合は以下のコマンドで起動済みコンテナとボリュームを破棄してください。
```bash
docker-compose down -v
```
